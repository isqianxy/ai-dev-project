# 测试策略（单元测试 / 集成测试）

## 1. 目标

- 覆盖推理循环、工具调用、记忆管理、权限确认、SSE 事件流等核心能力
- 建立可持续的回归能力与质量门禁

## 2. 单元测试（Unit）

- 推理引擎：终止条件、状态机转换、异常路径、资源约束
- 工具系统：注解扫描/注册、参数解析、权限校验、异常封装
- 记忆管理：滑窗裁剪策略、序列化/反序列化、TTL 行为

## 3. 集成测试（Integration）

- API：REST + SSE 的关键路径（断线重连、事件顺序、错误码一致性）
- 存储：MySQL/Redis 的读写一致性与过期策略
- RAG：Elasticsearch 索引写入、kNN/混合查询、**租户 filter 必填**、ES 不可用时的降级路径
- Human-in-the-loop：确认请求创建 → 用户确认 → 工具实际执行 → 事件回放

## 4. 质量门禁（建议）

- PR 必须通过：单元测试、核心集成测试、静态检查（后续在 `scripts/` 中落位）
