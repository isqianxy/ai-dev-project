# 推理引擎设计（Reasoning Engine）

## 1. 需求映射

- **ReAct 循环**：支持 Thought（思考）- Action（行动）- Observation（观察）的循环逻辑。
- **终止条件**：具备明确的“任务完成”或“无法解决”判定逻辑，防止无限循环。
- **多模型适配**：通过抽象模型层与配置切换不同模型提供方。

## 2. ReAct 状态机（抽象）

- **状态**：`THINK` → `ACT` → `OBSERVE` →（回到 THINK 或终止）
- **输入**：用户任务、当前上下文、历史 Observation、工具可用列表、权限上下文、资源限制策略
- **输出**：下一步 Action（调用工具/请求人工确认/直接回答/终止）

## 3. 终止条件（建议规则）

- **完成**：模型输出明确完成标记（如 `FINAL`）或编排器判定目标已达成。
- **不可解**：连续 N 次无法产生有效 Action；或出现不可恢复错误；或超出最大循环次数/时间预算。
- **安全中断**：需要人工确认但超时/拒绝；或触发权限/策略禁止继续执行。

## 4. 资源约束

- **循环上限**：最大步骤数（maxSteps）。
- **时间预算**：总任务超时时间（maxDuration）。
- **Token/上下文预算**：短期记忆滑窗策略，优先保留最近 Observation 与关键约束。

## 5. 错误与退避

- **模型错误**：重试（带指数退避）+ 熔断；记录失败原因并可视化。
- **工具错误**：分为可重试与不可重试；异常信息必须回传为 Observation，供下一步决策使用。
