# 工具系统设计（Tooling System）

## 1. 需求映射

- **声明式工具定义**：通过 Java 注解（如 `@Tool`）将 Service 层能力暴露为可调用工具。
- **参数自动解析**：将 LLM 生成的 JSON 参数自动映射为 Java POJO。
- **结果反馈**：工具执行结果（含异常）实时回传推理引擎作为 Observation。

## 2. 核心概念

- **Tool 定义**：名称、描述、参数 Schema、权限级别、幂等性/副作用标记、超时策略。
- **Tool Registry**：启动时扫描并注册工具；支持按配置启用/禁用。
- **Tool Executor**：统一执行入口，负责参数解析、权限校验、超时与隔离、异常封装与事件上报。

## 3. 参数 Schema 与解析

- **契约来源**：以 Java 类型定义生成（或维护）参数 Schema，供模型选择与构造参数。
- **解析策略**：
  - 严格 JSON 校验（字段类型/必填项）
  - 宽容策略（可配置）：允许额外字段但丢弃；或自动纠错并记录警告
- **错误处理**：参数不合法时返回结构化错误 Observation，禁止直接抛出未捕获异常导致任务中断。

## 4. 权限与 Human-in-the-loop

- **权限分级**：只读工具与写工具分级；写操作默认需要人工确认。
- **确认机制**：工具执行前生成“待确认请求”，由 API 层输出到前端并等待确认结果。

## 5. 可观测事件

- 事件类型建议：
  - 工具发现/注册
  - 工具调用开始/结束
  - 参数解析成功/失败
  - 权限校验拒绝/确认请求创建/确认结果
  - 异常与重试
