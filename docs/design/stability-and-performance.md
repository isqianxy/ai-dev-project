# 稳定性与性能设计

## 1. 需求映射

- **稳定性**：LLM 响应异常或工具调用超时需要优雅退避；引入熔断机制。
- **性能**：大数据量处理必须流式读取，严禁一次性加载进内存防止 OOM。

## 2. 超时与重试

- **模型调用**：超时 + 限次重试 + 指数退避
- **工具调用**：按工具配置超时；区分可重试与不可重试异常

## 3. 熔断与降级

- **熔断**：当模型/某类工具持续失败达到阈值，短时间内拒绝请求并返回可解释的失败原因
- **降级**：在可接受场景下返回摘要/部分结果，并明确告知未完成的原因

## 4. 流式处理与内存保护

- 对文件/数据库大结果集采用分页或游标式迭代
- 对 SSE 输出采用分片事件推送，避免一次性构建巨大响应体
- 对上下文窗口实施滑窗与摘要策略，控制上下文膨胀
