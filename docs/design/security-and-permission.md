# 安全与权限设计（含 Human-in-the-loop）

## 1. 需求映射

- 工具调用需要权限隔离
- 写操作必须支持人工确认（Human-in-the-loop）

## 2. 权限模型（建议）

- **主体**：用户/服务账号/系统任务
- **范围**：会话级（Session）、工具级（Tool）、资源级（Resource）
- **等级**：
  - 只读：默认允许（在策略与租户隔离下）
  - 写操作：默认禁止直接执行，必须进入确认流程

## 3. Human-in-the-loop 流程（建议）

1. 推理引擎产生写工具 Action。
2. 工具系统进行策略评估：是否需要确认、是否允许该主体提出确认请求。
3. 创建 `Approval` 请求并通过 SSE/REST 返回前端展示。
4. 用户确认后，才允许执行写工具；拒绝则返回拒绝 Observation 给推理引擎。

## 4. 防滥用与隔离

- 工具白名单与按环境启用（开发/测试/生产）
- 限流与配额：按用户/会话/工具维度
- 输入输出审计：关键字段脱敏与可追溯编号
