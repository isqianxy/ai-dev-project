# 可观测性设计（推理链路可视化）

## 1. 需求映射

- 每一轮“思考过程”必须记录日志
- 支持在前端展示推理链路

## 2. 事件与日志

- **事件优先**：将推理步骤与工具调用抽象为结构化事件，既可落库，也可用于 SSE 输出。
- **关联标识**：`traceId` / `sessionId` / `runId` / `stepId` / `toolCallId` 必须贯穿全链路。

## 3. 指标（建议）

- 任务成功率、平均耗时、步骤数分布
- 工具调用成功率、超时率、重试次数
- 模型调用耗时、失败率、熔断触发次数

## 4. 前端展示（建议）

- 以时间线展示：Reasoning Step → Tool Call → Observation → 决策结果
- 支持展开查看参数与结果（对敏感字段脱敏）
